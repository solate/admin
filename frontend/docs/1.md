项目目标： 请作为一个资深前端架构师，为我设计并实现一个现代化、美观、高效且可用于生产环境的后端管理系统前端界面。

一、 设计指南与视觉规范

设计系统参考： 深度参考 Ant Design 和 Element Plus 的设计哲学，融合其清晰、理性、克制的视觉风格。同时，吸收现代 SaaS 产品（如 Vercel Dashboard, Linear）的简洁感和空间感。

色彩体系：

主色： 采用一种科技蓝或商务蓝（例如 #1677ff 或 #1e88e5），用于主要按钮、关键高亮和导航激活状态。

中性色： 使用从浅到深的灰色阶（例如 #fafafa, #f0f0f0, #d9d9d9, #8c8c8c, #434343, #262626），用于背景、边框、文字。

功能色： 定义明确的成功绿（#52c41a）、警告黄（#faad14）、错误红（#ff4d4f）和信息蓝。

支持深色模式： 请同时提供一套适配的深色主题色板。

排版与字体：

使用无衬线字体族，如 -apple-system, BlinkMacSystemFont, ‘Segoe UI‘, Roboto, ‘Helvetica Neue‘, Arial, sans-serif。

建立清晰的层级：页面标题（24px/28px）、区块标题（16px/20px）、正文（14px）、辅助文字（12px）、标签文字（12px）。

间距与圆角：

采用 8px 基准网格。所有间距（内边距、外边距）应是 8 的倍数（8, 16, 24, 32…）。

组件圆角统一：小圆角（4px，用于按钮、输入框），中圆角（8px，用于卡片），大圆角（12px，用于模态框）。

二、 布局与组件设计

整体布局： 采用经典的左右布局。

侧边导航栏： 固定在左侧，宽度可收缩（Collapsible）。包含 Logo、主导航菜单（支持多级嵌套）、底部用户信息/设置入口。菜单项应有激活状态反馈和悬停效果。

顶部栏： 固定在上方，包含：面包屑导航、全局搜索框、通知中心、全屏切换、用户头像下拉菜单。

主内容区： 自适应宽度，具有舒适的左右内边距。背景为浅灰色（#f5f5f5）。

核心页面组件要求：

数据表格页：

顶部工具栏：包含“新增”、“批量操作”、“筛选/搜索”、“导出”等按钮。

表格：斑马纹、悬停高亮。列支持固定、排序、过滤。

分页器位于表格右下角。

行内操作列（查看、编辑、删除）采用图标按钮或“更多”下拉菜单。

表单页：

将表单字段分组在清晰的卡片容器中。

使用标签在上方的对齐方式，确保可读性。

区分主要操作（提交）和次要操作（取消、重置）。

对必填字段有明确标记（红色*）和验证提示。

详情/仪表盘页：

使用信息卡片（Card）进行数据分组展示。

合理利用统计数字、进度条、标签、徽标（Badge）等组件。

考虑嵌入简单的图表（如 ECharts 或 Recharts 风格的折线图/柱状图示意区）。

登录/404/403 页面： 设计简洁、专注的中心化页面，有适当的插画和明确的行动号召。

三、 交互与动效

加载状态： 所有异步操作（提交、加载数据）必须有明确的反馈。使用骨架屏（Skeleton Screen）替代传统的旋转加载图标，尤其是在表格和卡片区域。

悬停与反馈： 所有可交互元素（按钮、链接、菜单项）必须有平滑的悬停和点击状态变化。

过渡动画： 页面切换、模态框弹出、侧边栏收缩等操作，应伴有温和的淡入淡出或滑动过渡效果（持续时间 200-300ms）。

四、 开发实施规范

技术栈建议（请按此生成代码）：

框架： Vue 3 + Composition API （或 React 18 + Hooks，请根据我的下一个提示选择）

UI 组件库： 基于 Element Plus （Vue）或 Ant Design （React）进行开发与定制主题。

样式方案： 使用 Tailwind CSS 进行高效、原子化的样式编写，并覆盖组件库默认样式。

状态管理： Pinia （Vue）或 Zustand/Redux Toolkit （React）。

HTTP 客户端： Axios，并封装统一的请求/响应拦截器（处理 token、错误等）。

工具库： Day.js（日期）, Lodash-es（工具函数）。

代码质量：

组件应职责单一，可复用。

使用 TypeScript 确保类型安全。

注释关键逻辑。

实现响应式设计，确保在桌面端（≥1024px）和 iPad 横屏（≥768px）下有良好体验。
